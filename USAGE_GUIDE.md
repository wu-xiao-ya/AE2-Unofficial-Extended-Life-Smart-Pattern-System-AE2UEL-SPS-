# AE2UEL 智能模式系统 - 使用指南

## 快速开始

### 1. 获取智能模式物品
在创造模式物品栏的"工具"分类中找到"智能模式"物品。

### 2. 打开模式编辑器
右键点击智能模式物品打开GUI界面。

### 3. 编码模式
- **输入物品**: 将原料物品拖拽到左侧的输入槽位
- **输出物品**: 将产物物品拖拽到右侧的输出槽位
- **自动识别**: 系统会自动识别物品的矿物辞典条目

### 4. 应用模式
将编码完成的模式插入ME接口进行自动化生产。

### 5. 样板扩展器方块
放置样板扩展器方块并连接到AE网络后，它会自动扫描ME接口中的通配符样板并展开为多个独立配方。

> 只要矿物辞典存在对应条目（如 `ingotOsmium`、`plateOsmium`），会自动识别并加入。

## 模式类型

### 精确匹配
使用特定物品ID进行精确匹配。

### 矿物辞典匹配
匹配整个矿物辞典条目，例如：
- `ingotCopper` - 匹配所有铜锭
- `oreIron` - 匹配所有铁矿石
- `dustRedstone` - 匹配所有红石粉

### 处理模式
支持处理模式，用于自动化生产线。

## GUI操作

### 基本操作
- **拖拽**: 从物品栏拖拽物品到模式槽
- **Shift+点击**: 快速转移物品
- **关闭**: GUI关闭时物品自动返回物品栏

**模式无法插入ME接口？**
- 确保模式已正确编码
- 检查AE2UEL版本兼容性
- 确认ME接口有可用槽位

**GUI无法打开？**
- 检查物品栏中是否有智能模式物品
- 确保在游戏世界中（非菜单界面）

**物品丢失？**
- GUI关闭时物品会自动返回物品栏
- 如果物品丢失，请检查游戏日志

### 兼容性
- 需要AE2UEL v0.56.8+
- Minecraft 1.12.2
- Minecraft Forge 14.23.5.2847+
- CLR理论也可兼容，仅测试部分版本

## 技术说明

### 数据存储
模式数据通过NBT标签持久化保存，包括：
- 输入物品列表
- 输出物品列表
- 矿物辞典映射
- 处理模式设置

### 性能影响
- 匹配算法优化，不影响游戏性能
- 内存使用量小
- 支持大量并发模式

### 自定义配方映射 (config)
当 CRT 或其他脚本写法不规范时，可在配置中显式指定配方对，系统会优先使用这些配置：

1. 启动一次游戏，生成配置文件 `config/sampleintegration.cfg`
2. 打开配置文件，在 `pattern_expander` 类别下添加：

```ini
customRecipePairs = [
	"inogtCopper -> plateCopper",
	"ingotTin -> plateTin",
	"ingotBronze -> gearBronze"
]
```

说明：
- 左右两侧是明确的矿辞条目，不需要通配符
- 配置会先被读取，再进行矿辞扫描补全
- 支持不规范前缀（如 `inogt`）
